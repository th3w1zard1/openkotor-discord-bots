# HK-86 — Role Picker

HK-86 is the server's role assignment bot. It lets you pick "designations" (roles) that control which pings and topics you see. You do not need any special permissions — any server member can assign or remove roles from themselves.

---

## Quick Start

1. Type **`/designations panel`** to open the role picker.
2. Check the boxes for the roles you want. Uncheck the ones you do not want.
3. Click submit. All changes apply at once.

That is it. You can come back and change your picks any time.

---

## Commands

### `/designations panel`

Opens a private dropdown menu where you can check and uncheck all available roles at once. Your current roles show up as already selected. When you submit, everything updates in one step — roles you checked get added, roles you unchecked get removed.

Only you can see this menu.

### `/designations list`

Shows every available role organized by category, with a short description of each one. Only you can see this.

### `/designations assign`

Add one specific role to yourself.

| What to fill in | What it means |
|---|---|
| **designation** | The role you want (start typing and pick from the list) |

If you already have the role, nothing changes. Only you can see the response.

### `/designations remove`

Remove one specific role from yourself.

| What to fill in | What it means |
|---|---|
| **designation** | The role you want to remove (start typing and pick from the list) |

If you do not have the role, nothing changes. Only you can see the response.

### `/designations reactions help`

Operators only — ephemeral checklist covering Developer Portal intents, **recommended bot invite URL** (permissions pre-filled), JSON path, and verification steps. Anyone can run it (no Manage Server required) so staff can paste the invite link and file path during setup.

### `/designations reactions status`

**Requires Manage Server.** Forces a reload of `reaction-role-panels.json` from disk and summarizes each configured panel (channel id, message id, mapping count, announce mode). Use after editing the JSON or when debugging “reactions do nothing.”

## React-for-role panels

Some channels may have a **pinned or guide message** where you pick roles by **adding or removing emoji reactions**:

- **Add** the emoji → HK-86 assigns the mapped role (if you do not already have it).
- **Remove** the emoji → HK-86 removes that role (if you had it).

Each time you toggle, the bot may **reply** under that message (or DM you, depending on setup) with a short **HK-style** confirmation — the exact line is picked at random but stays relevant to the role.

Only reactions on **configured** messages and **configured** emojis do anything. Anything else is ignored.

## Operators: reaction panel configuration (no restart)

The bot watches a JSON file on disk and reloads it when the file changes (no redeploy or process restart):

- **Path:** `{HK_DATA_DIR}/reaction-role-panels.json` (default `HK_DATA_DIR` is `data/hk-bot`).

### Discord Developer Portal

- **Privileged intents:** enable **Server Members Intent** (required for role assignment and member resolution). Presence and Message Content are optional for react-for-role; Message Content is **not** required.
- **Guild Messages** and **Guild Message Reactions** are **non-privileged**: HK-86 requests them when connecting. There are **no separate Developer Portal toggles** for those two — configuration is entirely in bot code (already enabled in this deployment).

### Bot invite permissions

Use an invite that includes at least: View Channel, Send Messages, Embed Links, Read Message History, Add Reactions, Manage Roles, Use Slash Commands, and Use External Emojis (for custom emoji on panels). **`/designations reactions help`** prints a pre-built invite URL and permission integer `2416266304`.

### Example config in the repo

Copy [`apps/hk-bot/reaction-role-panels.example.json`](../../apps/hk-bot/reaction-role-panels.example.json) to `{HK_DATA_DIR}/reaction-role-panels.json` on the host and replace every placeholder snowflake with real IDs.

### Example `reaction-role-panels.json`

```json
{
  "version": 1,
  "defaultAnnounceMode": "reply",
  "replyCooldownMs": 3000,
  "panels": [
    {
      "channelId": "CHANNEL_SNOWFLAKE",
      "messageId": "MESSAGE_SNOWFLAKE",
      "announceMode": "reply",
      "mappings": [
        { "emoji": "✅", "roleId": "ROLE_SNOWFLAKE" },
        { "emoji": "myemoji:123456789012345678", "curatedRoleId": "reone" }
      ]
    }
  ]
}
```

| Field | Meaning |
|---|---|
| **defaultAnnounceMode** | `reply` (default), `dm`, or `silent` — how HK responds on success or failure for panels that omit `announceMode`. |
| **replyCooldownMs** | Minimum milliseconds between HK replies for the **same user** on the **same panel message** (default `3000`). Role changes still apply; only the chat reply may be skipped to reduce spam. |
| **panels[].channelId** / **messageId** | The Discord IDs of the reaction panel message. |
| **panels[].announceMode** | Optional override per panel. |
| **mappings[].emoji** | Unicode emoji character, **or** custom emoji as `name:snowflake` (same pattern Discord uses internally). |
| **mappings[].roleId** | Discord role ID. **Preferred:** survives role **renames** on the server. |
| **mappings[].curatedRoleId** | ID from the HK designation catalog (see `/designations list`). Resolves to the guild role whose **name** matches the catalog — same rules as slash commands (rename the Discord role away from the catalog name and this mapping breaks until names match again). |

Each mapping must have **exactly one** of `roleId` or `curatedRoleId`.

### Practical tips

- Copy **channel**, **message**, and **role** IDs from Discord (Developer Mode → copy ID).
- After editing the JSON, save the file; the bot picks up changes on the next reaction event (or run **`/designations reactions status`** to force a reload and confirm panels).
- Keep HK-86’s role **above** every managed role in Server Settings → Roles, same as for `/designations`.
- Redeploy slash commands once after upgrading HK-86 so **`/designations reactions`** appears (`pnpm dev:hk` / your process manager runs `deployGuildCommands` on startup).

## How the Bot Responds

HK-86 talks in character. Its responses have prefixes that tell you what type of message it is:

- **Statement:** — Confirmation that something worked (e.g., "Statement: Assigned reone, Watch Party.")
- **Observation:** — A warning about something unusual (e.g., "Observation: Missing guild roles for Script Slicer." — this means a server admin needs to create that role)
- **Mockery:** — An error with some attitude (e.g., "Mockery: Role hierarchy prevented access to Modzilla." — this means the bot does not have the right permissions to manage that role)

If you see an Observation or Mockery message, it is not your fault — let a server admin know so they can fix the role setup.

## FAQ

**Can I pick multiple roles?**
Yes. Use `/designations panel` to select as many as you want at once.

**Can I change my roles later?**
Yes. Run `/designations panel` again and update your selections. Or use `/designations assign` and `/designations remove` to add or drop one at a time.

**Do these roles give me any special powers?**
No. These are notification/topic roles only. They control which channels and pings you see. They do not grant moderation powers or special permissions.

**I got a "Missing guild roles" message. What do I do?**
That role has not been created on the server yet. Ask a server admin to create it with the exact name shown in the role list above.

**I got a "Role hierarchy" error. What does that mean?**
The bot's role is positioned lower than the role it is trying to assign. A server admin needs to move the HK-86 bot role higher in the role list (Server Settings → Roles → drag it up).

**How do react-for-role panels relate to `/designations`?**
They are two ways to toggle the **same kind** of self-assignable roles. Slash commands use the designation catalog; reaction panels use whatever mappings admins put in `reaction-role-panels.json` (often the same roles). Prefer **`roleId`** in JSON if Discord role names change often.
